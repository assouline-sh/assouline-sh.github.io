Foundations of Cybersecurity

2022-05-30

Foundations of Cybersecurity (CY2550) is the class I have been most excited about thus far. For once, instead of studying cybersecurity on my own and trying to figure out how to navigate the mass of information out there, I would be given structure- a clear learning path. Though I feel I didn’t learn a lot of new concepts, it was a great reinforcement of my previous knowledge. The genuinely fun assignments made me excited to continue progressing through the cyber program.

Below is an overview of my key takeaways from the course.

History

  • Cybersecurity arguably traces its origins to the 1970s, the golden age of phreaking
  • ARPANET → The Orange Book → Computer Fraud and Abuse Act → More secure protocols → Internet becomes privatized → Internet rapidly grows, is insecure, and unprepared for such a connected world, leading to the proliferation of several worms, viruses, and attacks

Threat Modeling

  1. Identify your assets
  2. Enumerate attack surfaces
  3. Hypothesize attackers and map them to:
    1. Desired value from assets
    2. Ability to target vulnerable surfaces from attack surfaces
  4. Survey mitigations
  5. Balance cost versus risk
  • Various methodologies for threat modeling are available, including: STRIDE, DREAD, OWASP testing framework, PASTA, VAST, and OCTAVE

Cryptography

  • Confidentiality, Integrity, Authenticity, and Non-repudiation, are often assured through cryptography
  • No security through obscurity; Kerchoffs’ Principle and Shannon’s Maxim
  • One time pads, symmetric key cryptography, block ciphers
  • Asymmetric cryptography, certificate authorities, Diffie-Hellman basis for protocols with Perfect Forward Secrecy, RSA based on difficulty of factoring the product of primes

Authentication

  • Classes of secrets: something you know (i.e. password), have (i.e. phone), or are (i.e. biometric)
  • Passwords are often weak enough for brute-force or dictionary attacks, and hash chains can enable time-space efficient reversal of hash functions (but is prone to collisions)
    • Rainbow tables improve on hash chains by using a family of reductions instead of a single reduction to reduce the likelihood of collisions; defeated by salting and key stretching
  • Measure password quality with entropy S = log₂N^L; note that length matters more than symbol types

Access Control

  • Discretionary access control vs. mandatory access control
  • With setuid, a program may execute as the file owner, rather than executing principal
    • Note: only setuid on compiled binary programs, not scripts, as it can lead to TOCTOU vulnerabilities
  • A capability-based access control system can fix the confused deputy problem (subject cannot tell which principal it is serving when it performs a write) because selecting a capability also selects a master
  • Security models: Bell-LaPadula (read down, write up) only address confidentiality while Biba Integrity Model (read up, write down) only addresses integrity

Social Engineering

  • Humans are the weakest link in the cybersecurity chain
  • Subconscious decision-making reduces cognitive burden, like subconscious cognitive biases including behavioral, social, and memory biases
  • Social engineering attacks include: baiting, tailgating, scareware, advance-fee scams, phishing or spear phishing
    • Leverage cognitive biases and rely on: information asymmetry, context construction, elicitation. and persuasion

Systems Security

  • Hardware support for process isolation with virtual memory and protected mode execution (rings) which stops direct access to devices and allows OS to impose access control checks
    • Base and bound registers, segmentation, and page tables enforce virtual memory
    • Ring 0: OS → Ring 1, 2: device drivers → Ring 3: userland
  • File access control, anti-virus, firewall, and secure logging
  • Separation of privilege, least privilege, compromise recording, work factor, authentication rate limiting, open design, economy of mechanism, and complete mediation

Exploits

  • Cross site scripting can be reflected, stored, or DOM-based, and allows user input to be interpreted as document structure to lead to malicious code execution
    • To mitigate: cookie restrictions, client-side filter, input validation, output filtering
  • SQL injections
    • To mitigate: escaping inputs, sanitizing inputs, prepared statements, object-relational mappings, domain-specific languages
  • In buffer overflow, vulnerability often stems from the fact that low-level languages are not memory safe and control flow information is stored inline with user data on the stack
    • Compiled shellcode that is injected must be zero-clean, and an NOP sled is often built in front of the shellcode
    • To mitigate: stack canaries (sentinel values before each saved IP), non-executable stacks, address-space layout randomization

DDOS

  • Denial of service achieves scale with volunteers, reflection and amplification, and botnets
  • Smurfing works because ICMP protocol doesn’t include authentication and the attacker benefits from an amplification factor
  • To mitigate: anti-amplification, network filters, anti-spoofing, content distribution networks,

APT

  • Generally, cybercriminals are profit-drive, opportunistic, and part of an underground economy while APTs are intelligence and espionage-driven, highly targeted, and highly secretive
  • Lazarus group, PLA Unit 61398, Fancy Bear, Cozy Bear
  • Shadow brokers, Vault7, Stuxnet
  • Tallinn manual examined how existing international law applies to online conflict
  • Hoarding vs. disclosing zero-days and Vulnerabilities Equities Process

Cyberlaw

  • Computer Fraud and Abuse Act (CFAA) of 1986 prohibits accessing a protected computer without authorization; authorization can be defined by policy or by mechanism
  • Digital Millennium Copyright Act (DMCA) of 1998 criminalizes circumvention of access controls and the distribution of circumvention tools, regardless of infringing on copyright
  • CAN-SPAM Act of 2003 enshrines an opt-out, rather than an opt-in system

Web Privacy

  • Online advertising companies, online data brokers, and offline data brokers track you
  • Stateful identifiers (IP address, cookies, locally shared objects (LSOs), E-tags, etc) and stateless fingerprints (browser and OS version, plugins / extensions, audio and battery, etc)
  • To mitigate tracking: incognito browsing, VPN, secure browser, delete cookies, localstorage, and cache, ad and tracker blocking extensions, disable JavaScript, uMatrix extension
  • Subject Access Rights give the ability to see, modify, and delete data collected about you while Affirmative notice and consent is prominent disclosure of collection and ability to opt-out
